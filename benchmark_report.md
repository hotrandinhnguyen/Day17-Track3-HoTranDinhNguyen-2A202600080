## Key Findings

1. **Cross-session retention**: Memory agent correctly recalls user preferences (e.g., Python preference, response style) in new sessions without re-asking.

2. **Episodic proactivity**: When a user was previously confused (e.g., async/await), the memory agent automatically adds extra explanation in follow-up sessions.

3. **Semantic retrieval**: ChromaDB enables the agent to surface thematically related past exchanges even when the user doesn't explicitly reference them.

4. **Context overhead trade-off**: Memory adds ~129 chars of context per turn. This cost is amortised by avoiding repeated user-preference questions across sessions.
